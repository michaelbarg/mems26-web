## Q5: Root Cause

"LIVE SETUPS appears to grow upward because the browser's
default scroll anchoring keeps the BOTTOM of the scroll
container visible when new content is added. The selected
sweep detail card (~400px) + BuildingNow + LIVE SETUPS
exceeds viewport height. When LIVE SETUPS items populate,
the scroll position stays at the bottom, making it LOOK like
content above disappeared — but it's just scrolled off the top."

NOT a flexDirection bug. NOT a CSS issue. It's scroll anchoring.

## Fix

Add `overflowAnchor: 'none'` on the scroll container to disable
browser scroll anchoring. This tells the browser NOT to adjust
scroll position when content size changes. Combined with the
existing scroll-reset on tab change, the top stays anchored.
