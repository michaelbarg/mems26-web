## Step 0.3 — Root Cause of "Dispatch Not Working"

The dispatch chain code is correct and intact. The most likely cause:

1. **Bridge was not running** when EXECUTE was clicked (Bridge is local on Mac)
2. **Bridge config not yet published** — first publish happens on startup, but if Bridge started after backend cold-start, the `mems26:bridge_config` key might not exist, causing backend to default to STRICT and reject outside killzone
3. **Render cold start** — backend returns 502 on first request after sleep, EXECUTE fails silently
