**File:** `backend/main.py` lines 1721-1726

    t1 = round(entry + T1_MIN_PT, 2) if direction == "LONG" else round(entry - T1_MIN_PT, 2)

### The 7147.50 Mystery — SOLVED

Setup: SHORT @ entry=7157.50, stop=7161.25, risk=3.75pt

1. Backend receives t1=7153.75 (C1 from frontend, correct)
2. Backend OVERWRITES: `t1 = 7157.50 - 3.75 * 1.5 = 7151.875`
3. Then checks: `abs(7151.875 - 7157.50) = 5.625 < T1_MIN_PT(10.0)`
4. Override: `t1 = 7157.50 - 10.0 = 7147.50`

**7147.50 = entry - T1_MIN_PT = 7157.50 - 10.0**

### Impact
- DLL receives t1=7147.50 instead of C1=7153.75
- DLL sends bracket with target @ 7147.50 (10pt away)
- Sierra shows "B Lmt 7147.50" which matches nothing the user expected

### Fix
Remove the target recalculation. Frontend already computes correct
C1/C2/C3 per the setup. Backend should forward them as-is.
