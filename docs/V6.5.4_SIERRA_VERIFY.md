# Sierra Dispatch Verification — Local Test

Run these 3 steps when Bridge is running locally.

## Step 1: Test dispatch endpoint
```bash
curl -X POST https://mems26-web.onrender.com/trade/test-dispatch \
  -H 'content-type: application/json' \
  -d '{}'
```
Expected: `{"ok": true, "redis_key": "mems26:trade:command:test", ...}`

## Step 2: Check Bridge log
```bash
tail -f /tmp/bridge.log | grep -i "C4\|command\|dispatch"
```
Expected: `[C4] written: TEST_DISPATCH ...` within 2 seconds.

## Step 3: Check file on disk
```bash
ls -la /Users/michael/SierraChart2/Data/trade_command.json
cat /Users/michael/SierraChart2/Data/trade_command.json
```
Expected: JSON with `"cmd": "TEST"` and recent timestamp.

If Step 1 works but Step 2 doesn't show anything:
- Bridge is not running, or CLOUD_URL is wrong
- Check: `echo $CLOUD_URL` in Bridge terminal

If Step 2 shows the command but Step 3 file is missing:
- SC_COMMAND_PATH is wrong
- Check: `echo $SC_COMMAND_PATH` in Bridge terminal
